Privacy Policy — AskWispr

Last updated: 24 April 2026

AskWispr is a software platform operated by Vidyaax ("Vidyaax", "we", "us"). The platform lets businesses ("Merchants") run multilingual, AI-assisted customer-support assistants on WhatsApp and similar channels (the "Platform"). Our first Merchant is ; more will follow.

This policy explains how Vidyaax handles personal data in two distinct roles:

  1. As a data processor, when we process data about a Merchant's end customers (the people who message the Merchant on WhatsApp) on the Merchant's behalf.
  2. As a data controller, for operational data we hold about Merchant users (admins, developers) and for the limited platform telemetry we control directly.

If you are an end customer who messaged a Merchant (e.g. Jewellers) on WhatsApp and reached AskWispr's infrastructure, the Merchant is the primary data controller for your conversation; please also consult that Merchant's privacy notice. Vidyaax acts on the Merchant's instructions.

2. Data we process on behalf of Merchants (processor role)

When an end customer messages a Merchant's WhatsApp number, the Platform receives and processes:

Data Source Purpose
WhatsApp phone number (wa_id, E.164) Meta WhatsApp Cloud API Route the reply; deduplicate Meta retries; scope short-term conversation memory
Wispr Meta WhatsApp Cloud API Optional personalised greeting
Message content (text, interactive button replies) The end customer Classify intent, retrieve relevant FAQ / catalog answers, generate the Merchant's reply
Message metadata (message.id, timestamp, detected language) Meta WhatsApp Cloud API Idempotency, diagnostics, language detection
Short-term conversation state (detected language, last intent, last catalog query, summary of recent turns) Derived by the Platform Multi-turn follow-ups (e.g. "show more")

We do not collect payments, order history, government IDs, precise location, contacts, device identifiers, or any special-category data. If an end customer volunteers such information, it is treated as message content under the retention rules in Section 4.

Media messages (image, audio, document, location, contact, sticker) are not downloaded or processed in v1 — the Platform replies that media is not yet supported.

Each Merchant's data is logically segregated. Vidyaax does not combine one Merchant's end-customer data with another Merchant's.

3. Data we process as a controller

  • Merchant admin accounts. Name, work email, role, authentication material — to provide the Platform to that Merchant.
  • Aggregated, non-identifying telemetry. Counts of messages handled, latency percentiles, intent-classification accuracy, language distribution — to operate, secure, and improve the Platform.
  • Operational logs. Described in Section 4.

4. Retention

  • Conversation memory (detected language, recent turns, summary): held in server memory only and automatically discarded 60 minutes after the end customer's last message, or immediately on service restart. There is no database of chat history.
  • Idempotency cache (WhatsApp message.id values, no message content): bounded in-memory LRU (~10,000 entries), evicted as new messages arrive.
  • Operational logs (timestamp, wa_id, detected intent/language, diagnostic text): retained up to 30 days for debugging and abuse prevention, then deleted.
  • Merchant admin records: retained for the life of the Merchant relationship plus up to 12 months after termination, then deleted or anonymised.
  • Meta WhatsApp Cloud API retains message metadata under its own retention rules — see https://www.whatsapp.com/legal/privacy-policy.

Vidyaax does not build long-term end-customer profiles. Vidyaax does not use end-customer messages to train AI models.

5. Legal basis / grounds for processing

  • Merchant's instructions under a Data Processing Agreement (DPA) for end-customer data we process on their behalf.
  • Contract with the Merchant for admin-account data.
  • Legitimate interests (GDPR Art. 6(1)(f)) / legitimate uses (DPDP Act 2023 §7) for platform security, fraud prevention, and service improvement.
  • Consent — when an end customer initiates a WhatsApp conversation with a Merchant, that interaction constitutes consent to the automated handling described here. It can be withdrawn (Section 8).

A current list of sub-processors is available at https://askwispr.com/legal/subprocessors. We will notify Merchants in advance of any material change so they can object.

Vidyaax does not sell or rent personal data. Vidyaax does not use end-customer messages to train models, for advertising, or for any purpose other than answering that customer's question on behalf of the Merchant.

Cross-border transfers: where a sub-processor processes data outside India, we rely on Standard Contractual Clauses (or equivalent) and send only the minimum payload needed to answer.

7. Security

  • Webhook integrity: every inbound Meta callback is verified via HMAC-SHA256 using the Merchant's app secret before it is processed.
  • TLS in transit for all connections between Meta, the Platform, LLM providers, and Qdrant.
  • Secrets (access tokens, API keys) are held in a secrets manager / env vars, never in code or logs.
  • Least-privilege access to Merchant data; audited.
  • Tenant isolation at the data and retrieval layers.

No system is perfectly secure. If we become aware of a personal-data breach we will notify affected Merchants without undue delay and cooperate with statutory notifications (CERT-In, supervisory authorities) as required.

8. Your rights

Subject to GDPR, India's DPDP Act 2023, and other applicable law, end customers and Merchant users can request:

  • Access — a copy of the data Vidyaax currently holds about you.
  • Correction — fix inaccurate data.
  • Erasure / Deletion — see our Data Deletion Instructions.
  • Withdrawal of consent — stop messaging the Merchant, or block the Merchant's WhatsApp number.
  • Grievance redressal — contact our grievance officer. Indian users may also approach the Data Protection Board once constituted.

End customers: because the Merchant is the primary controller of your conversation data, requests are usually fastest when directed to the Merchant. Vidyaax will forward your request to the relevant Merchant if you contact us directly. To exercise a right that Vidyaax controls, email privacy@vidyaax.com or send DELETE over WhatsApp to the Merchant you interacted with.

We will respond within 30 days.

9. Children

The Platform is intended for users aged 18 and above. Vidyaax does not knowingly process data of minors. If a minor's data has reached the Platform, contact privacy@vidyaax.com and we will erase it.
